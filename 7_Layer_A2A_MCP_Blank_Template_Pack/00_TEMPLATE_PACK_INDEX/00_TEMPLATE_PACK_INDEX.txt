7-LAYER A2A AND MCP BLANK TEMPLATE PACK

Purpose
- This folder contains blank project-local templates for A2A Agent Card work and MCP specification work.
- It is intended for the 7-layer architecture phase and related multi-agent planning.
- It is separate from the shared Desktop Codex source library.

What is included
- 01 A2A Agent Card Template
- 02 A2A Signed Agent Card Template
- 03 MCP Server Specification Template
- 04 MCP Interface Inventory Template
- 05 MCP Security And Transport Template

How to use this pack
1. Start with the A2A Agent Card template if you need agent discovery metadata.
2. Use the signed Agent Card template if the agent identity must be signed or verified.
3. Use the MCP Server Specification template to describe the MCP server at a planning level.
4. Use the MCP Interface Inventory template to list tools, resources, prompts, and other exposed surfaces.
5. Use the MCP Security And Transport template to document transport, session, auth, and boundary rules.

Important source note
- The A2A Agent Card templates are based directly on the current public A2A protocol structure.
- MCP currently has an official spec and schema, but not a single official blank planning form in the same style.
- The MCP templates in this pack are spec-derived project templates created for planning and documentation clarity.

Primary reference sources
- A2A GitHub: https://github.com/a2aproject/A2A
- A2A proto: https://github.com/a2aproject/A2A/blob/main/specification/a2a.proto
- MCP GitHub: https://github.com/modelcontextprotocol/modelcontextprotocol
- MCP docs: https://modelcontextprotocol.io

Template conventions
- Keep placeholders in curly braces until the project-specific values are known.
- Replace placeholders deliberately.
- Keep one package focused on one document type.
