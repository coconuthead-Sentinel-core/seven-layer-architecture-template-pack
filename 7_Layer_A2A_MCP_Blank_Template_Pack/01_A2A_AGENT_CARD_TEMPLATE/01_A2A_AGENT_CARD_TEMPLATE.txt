A2A AGENT CARD TEMPLATE

Use for
- Defining the public discovery metadata for an A2A-compatible agent.
- Publishing identity, interfaces, capabilities, media types, and skills for agent-to-agent communication.

Important note
- This template is based on the current public A2A specification and proto field structure.
- Validate the final JSON against the current A2A spec before production use.

Template metadata
- Project name: {project_name}
- Agent name: {agent_name}
- Owner: {owner}
- Date: {date}
- Status: {draft|review|approved}

Field purpose guide
- name:
  Human-readable agent name.
- description:
  What the agent does and when another agent should use it.
- supportedInterfaces:
  Where and how this agent can be reached.
- capabilities:
  Streaming, push, extended-card, and extension behavior.
- securitySchemes:
  Auth methods supported by this agent.
- securityRequirements:
  What auth/scopes are required to call the agent.
- defaultInputModes / defaultOutputModes:
  Media types used by default.
- skills:
  The focused capabilities this agent exposes.

Suggested use in the 7-layer system
- Use this for Layer 5 peer-agent discovery and coordination metadata.
